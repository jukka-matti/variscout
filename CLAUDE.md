# VariScout

Structured investigation for process improvement — question-driven, evidence-based, AI-assisted. Offline-first, browser-based.

## Principles ([full constitution](docs/01-vision/constitution.md))

- **Journey-driven variation analysis** — FRAME → SCOUT → INVESTIGATE → IMPROVE; discovery, understanding, verification
- **Same analysis everywhere, AI optional** — PWA and Azure share identical analytical power; CoScout adds depth
- **Customer-owned data** — browser processing; when data moves (Teams, OneDrive, AI) it stays in customer's tenant
- **Question-first investigation** — questions before theories, multiple suspected causes (Turtiainen 2019)
- **Deterministic first, AI enhances** — statistical engine is authority, CoScout adds context
- **Three evidence types** — data (auto η²), gemba (go-see + photos), expert knowledge

## Key Patterns

- **No Backend**: All processing in browser, data stays local
- **Shared Logic**: Statistics in `@variscout/core`, charts in `@variscout/charts`
- **Props-based Charts**: Chart components accept data via props (not context)
- **Persistence**: PWA = session-only (no persistence); Azure Standard = IndexedDB (local); Azure Team = IndexedDB + Azure Blob Storage sync
- **Offline-first**: PWA works without internet after first visit
- **Zustand-First State**: `@variscout/stores` holds 4 domain stores (project, investigation, improvement, session) as the source of truth — no DataContext, no facade. Components read via direct selectors (`useProjectStore(s => s.field)`) and 7 derived hooks (`useFilteredData`, `useAnalysisStats`, etc.). Azure feature stores (`features/*/`) remain for UI-specific state. Persistence via `useProjectActions` (document persist) + sessionStore middleware (auto-persist).
- **panelsStore fields**: `highlightedFactor`, `investigationViewMode` ('map'|'findings'), `factorPreviewDismissed`, `activeImprovementView`, `highlightedIdeaId` (last two moved from deleted improvementStore, Apr 2026)
- **Strategy Pattern (ADR-047)**: `resolveMode()` + `getStrategy()` in `@variscout/core/strategy` — eliminates cascading mode ternaries. Each mode carries chart slots, KPI type, report config, metric labels, AI coaching hints. CoScout receives mode-specific methodology coaching (lean for yamazumi, multi-channel for performance).
- **Modular CoScout Prompt Architecture**: `assembleCoScoutPrompt()` in `packages/core/src/ai/prompts/coScout/index.ts` is the current entry point — returns `CoScoutPromptTiers` (tier1Static, tier2SemiStatic, tier3Dynamic, tools). Replaces the deprecated `buildCoScoutSystemPrompt()` flat-string function (still in `legacy.ts` for test compat). Sub-directories: `role/`, `phases/` (per journey phase), `modes/` (per analysis mode), `context/` (investigation + data + knowledge formatters), `tools/` (25-tool typed registry with `getToolsForPhase()`). See [CoScout Prompt Architecture](docs/05-technical/architecture/coscout-prompt-architecture.md).
- **Canonical type ownership**: UI presentation types (`DisplayOptions`, `HighlightColor`, etc.) live in `@variscout/core/ui-types`. Evidence Map types (`FactorNodeData`, etc.) live in `@variscout/core/evidenceMap`. Both re-exported by hooks and charts for backward compatibility.
- **Unified popout sync**: All pop-out windows (Findings, Improvement, Evidence Map) use `usePopoutChannel` from `@variscout/hooks` — BroadcastChannel for ongoing sync, localStorage hydration for initial state. Typed message contracts in `popoutMessages.ts`
- **FindingSource is a discriminated union**: Narrow before accessing `.category` (`'category' in src`). Variants: boxplot/pareto (category), ichart (anchorX/Y), yamazumi (category + activityType), coscout (messageId)
- **Feature store naming**: Azure feature stores use `*FeatureStore` suffix where needed (`useInvestigationFeatureStore`). The improvement **feature** store was deleted Apr 2026 — its UI state moved to `panelsStore`. The domain `improvementStore` in `@variscout/stores` remains. Remaining feature stores: `panelsStore`, `findingsStore`, `useInvestigationFeatureStore`, `aiStore`
- **`wouldCreateCycle`**: Lives in `@variscout/core/stats` (`packages/core/src/stats/causalGraph.ts`), not findings

See `.claude/rules/` for code style, chart, testing, and monorepo conventions.
