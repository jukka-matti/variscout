# View: `altitude-model`

The product-wide organizing frame (first-session spec §1): every surface is a level of one investigation. Not a screen itself — the map the screens live on. Interactive wireframe: [`assets/altitude-model.html`](assets/altitude-model.html).

```
 framing feeds                 the altitude axis              the way back up
 ─────────────                 ─────────────────              ───────────────
 b0 picker gives L1 its Y  →   L1 · System (the Y, health)
                                  ▼ zoom
 StepTimings/CCJ zones     →   L2 · Process (flow, bottleneck)
 materialize L2                   ▼ drill
 step-bound Y/X (CCJ)      →   L3 · Step (local mechanism)    ▲ findings file to their
                                  ▼ "See the data" (scoped)      step (processLocation)
                               Explore (the microscope)        ▲ minted factors join
                                  ▼ brush/pin/point/signal       the X list
                               the condition (the Finding)     ▲ the Wall explains
                                                                 (derived edges, CS-12)
```

- **Landing rule:** land at the highest altitude the data has established (no Y → b0; Y only → L1; map → L2; drilled → L3).
- **One grammar at every altitude:** engine proposes (chips/banners), analyst decides — Y inference, pair detection, mode detection, shift signals, best-subsets.
- **Click assigns a role; drag assigns a place** — same chips, gaining geography as structure appears.
- The wrapper around the whole descent: the **Untitled project** (always exists; ceremony opt-in).
