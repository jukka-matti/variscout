# F3 Data-Flow Foundation: PWA Dexie normalized schema cutover (branch `data-flow-foundation-f3`, PR pending)

Atomic swap of `PwaHubRepository.dispatch` from blob-write to normalized-table-write. **Three locked scope decisions** from brainstorm: (1) **DB rename, not version chain** — new IDB database `variscout-pwa-normalized`, `version(1)` of new schema with no `.upgrade()` callback (Dexie can't redefine in-flight `version(1)`; renaming is spec-faithful + rollback-safe + acceptable pre-production cost of orphaned `variscout-pwa` on dev machines, mitigated by best-effort fire-and-forget `Dexie.delete('variscout-pwa').catch(() => {})` on construction). (2) **Narrow action coverage** — only `HUB_*`/`OUTCOME_*` handlers get real Dexie writes (plus `HUB_PERSIST_SNAPSHOT` decompose into hubs+outcomes+canvasState in a single transaction with stale-row cleanup preserving F2's blob-replacement invariant); `EVIDENCE_*`/`INVESTIGATION_*`/`FINDING_*`/`QUESTION_*`/`CAUSAL_LINK_*`/`SUSPECTED_CAUSE_*`/`CANVAS_*` stay no-op until F3.5 (evidence) and F5 (rest). (3) **canvasState as separate table** — `hub.canonicalProcessMap` decomposes out of the hub blob into a dedicated `canvasState` row (1:1 with hub) on save; reads join it back into ProcessHub shape via private `joinHub` helper inside a `db.transaction('r', ...)` wrap. **Plan-reality deltas absorbed:** (1) P1+P2+P3+P4 consolidated into one implementer dispatch because TypeScript compilation requires the schema swap + applyAction signature change + repository rewrite + opt-in extraction to land atomically; (2) `HUB_UPDATE_GOAL`/`HUB_UPDATE_PRIMARY_SCOPE_DIMENSIONS` now loud-fail on missing hub (matches `OUTCOME_ADD` invariant; tightens the F2 silent-no-op behavior); (3) `evidenceSources.getCursor` post-filters by `hubId` after sourceId index lookup (PWA schema is `'&id, sourceId'` — primary key on `id`, indexed by `sourceId`); (4) `--vrs` v1.1 bump deferred to F4 (in-memory `ProcessHub` shape unchanged, current v1.0 round-trips cleanly through `HUB_PERSIST_SNAPSHOT`). **F3.5 reconciliation point flagged:** PWA `evidenceSourceCursors: '&id, sourceId'` uses single primary key + post-filter; Azure `evidenceSourceCursors: '[hubId+sourceId]'` uses compound primary key. Both honor spec D3 (which prescribed each separately) but F3.5 ingestion writers must branch between apps when writing cursors (id-keyed put vs compound-keyed put). **Test count delta:** PWA suite 298 → 269 passing (-29 net); 95 new behavioral tests added; 124 pre-F3 cascade-helper tests legitimately removed alongside the helpers themselves (the `archiveDescendantsOfKindInDraft` / `cascadeArchive*InDraft` Immer placeholders are gone from `applyAction.ts`). Read-transaction wrap, atomicity (transaction rollback), loud-fail, stale-row cleanup, and getCursor cross-hub isolation all pinned with regression tests. **State:** Code review passed (Opus final review, 0 Critical, 1 Important = this entry, 3 Minor pre-existing/polish).
