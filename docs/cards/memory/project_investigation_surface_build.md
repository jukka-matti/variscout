## Method lessons (IM-1 run)
- **Layer pipeline works:** one focused dispatch per package, verify `tsc` src-clean, COMMIT each layer; parallelize across DISJOINT packages (ui+data+i18n ran concurrently). Never commit while a background agent edits (lint-staged `git stash` collides) — wait for all idle, commit per-package.
- **Vitest ≠ verification:** the 6 test agents ran vitest only on files they CHANGED, so compile-clean-but-runtime-fail tests in UNCHANGED files slipped through (processHub zeroed-questions, FrameView, schema.v7, ColumnMapping, StageFiveModal). The FULL `pnpm test` turbo run (incl apps) is the real gate. Also `tsc --noEmit` per package must be re-run after test edits (pwa build-tsc caught a FrameView mock-type the agent's vitest missed).
- **Adversarial review earns its keep:** the 4-dimension panel + synthesis caught a genuine behavior regression (searchProject relevance ranking collaterally deleted in the drop-Question commit, tests gutted to "input order") that the green gate could NOT — plus 2 live CoScout tool survivors. Always run it before a large-cascade merge.
- **Classify "pre-existing" by git, not by agent label:** `git log origin/main..HEAD -- <test> <src>` — unchanged test+src that fails = genuinely pre-existing (confirmed by running on a main worktree). Fix pre-existing on main FIRST for a clean baseline before a parallel wave.
- Agents can crash mid-run (socket error) — verify the working tree (tsc + greps) before trusting/committing a crashed agent's output.
- `gh pr merge --delete-branch` errors locally ("main already used by worktree") but the REMOTE merge succeeds — verify via `gh pr view N --json state`.
- **CROSS-WORKTREE CONTAMINATION (cost a real recovery, 2026-05-30):** NEVER run `git commit` (which triggers the lint-staged `git stash`/restore pre-commit hook) while ANY background implementer agent is actively writing — even in a DIFFERENT worktree. PR #250's polish commit collided with the still-running IM-7 implementer; IM-7's `predicates.ts`/`ProjectsTabView` edits leaked into the legacy worktree and `predicates.ts` got swept onto main (RED — it referenced `collaboratedAt` before the type had it). RULE: gate/commit/merge ONLY when all writer agents are idle. Merges themselves (push + `gh pr merge`, no local commit) are safe — only `git commit` triggers lint-staged. Audit all worktrees' `git status` before/after a commit when agents have been running.
- **Adversarial-review harness pitfalls:** (a) Workflow `args` does NOT reliably reach the script via `{scriptPath, args}` — the review silently targeted the stale `.claude/worktrees/im-0a-project-hub-1to1` cwd (its branch `im1-housekeeping` is a no-op ancestor of main) → empty diff → false "block". Use INLINE review scripts with HARDCODED worktree paths + a STEP-0 guard: `git -C <wt> branch --show-current` MUST equal the expected branch or ABORT. (b) Subagents default to that stale cwd — every git command needs `git -C <wt>` + absolute paths. (c) The Workflow validator rejects any script whose SOURCE contains the literal tokens `Date.now`/`Math.random`/`new Date` even inside prompt prose — write "wall-clock"/"RNG" instead.
- **The stale `.claude/worktrees/im-0a-project-hub-1to1` worktree (branch `im1-housekeeping`, already merged) is the default subagent cwd and a recurring footgun** — its branch is a no-op ancestor of main; safe to `git worktree remove` once the controller session no longer needs it as cwd.
- **Adversarial review keeps earning its keep:** caught IM-2's unsubmittable form (FORM_H foreignObject clip, invisible to jsdom) and IM-7's PWA hidden-vs-disabled signoff (a §9.2 contract violation + doc/test/comment contradiction) — both gate-invisible. Run it before every merge.
