PWA is "same analysis everywhere, AI optional" — NOT "the struggle is the point."

**Why:** The user clarified that PWA should have almost all the same features as Azure but without AI. The "guided frustration" framing was wrong for PRODUCT positioning — it implied PWA is deliberately crippled.

**How to apply:** When building features, default to making them available in both PWA and Azure. Only gate on: CoScout AI, cloud features (Teams, OneDrive, persistence), and infrastructure limits (factor count, row limits). Never gate core analysis capability.

**Nuance (Apr 1 2026):** "Guided frustration" is still valid as a PEDAGOGICAL concept in educational docs (docs/01-vision/philosophy.md, docs/02-journeys/case-based-learning.md). It describes the Sock Mystery teaching methodology — let students fail so they ask why. But it must NOT be used to describe the PWA product itself.
