**8a architectural shape (delivered, durable):**

1. **DMAIC charter correction** — Project Charter is the FIRST step of Define phase, not a late workflow gate. Sources: SixSigma Institute, DMAIC.com, SixSigma.us, GoLeanSixSigma. See `feedback_verify_methodology_before_gating` for the generalized rule.
2. **Tier reframe** — All five response paths free-tier-active in PWA + Azure per `feedback_pwa_philosophy`. Team-collaboration features (signoff, audit trail, alerts, RACI) tier-gate INSIDE each surface, not on the response-path button. See `feedback_tier_gate_inside_surface` for the generalized rule.
3. **2-state CTA machine** — `computeCtaState` in `packages/ui/src/components/Canvas/internal/responsePathCta.ts` returns `{ kind: 'active' } | { kind: 'prerequisite-locked'; reason } | { kind: 'hidden' }`. The 4-state machine (`active` / `tier-locked` / `hub-immature` / `awaiting-impl`) the original plan proposed never shipped. `hidden` is reserved for unwired handlers (don't tease unfinished features); `prerequisite-locked` covers the realistic mid-workflow state.
4. **Per-path readiness predicates** — `responsePathReadiness.ts` in `@variscout/core` exports `WorkflowReadinessSignals { hasIntervention, sustainmentConfirmed, isDemo? }` + `isCharterReady` (always true) / `isSustainmentReady` (needs intervention) / `isHandoffReady` (needs sustainment-confirmed). `computeHubMode` (cadence/first-time/demo enum) was rejected — it conflated tier and workflow position. See `feedback_hidden_vs_disabled_cta` for the philosophy.
5. **Stub destinations** — Charter / Sustainment / Handoff panels ship as one-paragraph pedagogical placeholders so buttons never go dead. PWA: `apps/pwa/src/components/{Charter,Sustainment,Handoff}Panel.tsx`. Azure: `apps/azure/src/components/{charter,sustainment,handoff}/{Charter,Sustainment,Handoff}Panel.tsx`. Routed via `usePanelsStore.show{Charter,Sustainment,Handoff}` actions in both apps.
6. **Charter entity location locked** — `ProcessHub.charters: Charter[]` (hub-level, multiple per Hub per Q1). Type will live in `@variscout/core/processHub.ts` once the form ships. F4 Document layer. **8a did NOT ship the Charter type or table** — only committed to the data-model location.
