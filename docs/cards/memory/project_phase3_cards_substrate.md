## Lifecycle rules

**Cards are append-only by definition.** To change a card:
1. **Typo / banner fix**: edit in place. Defensible because semantic content unchanged.
2. **Semantic change**: write NEW card with `supersedes: [<prior-id>]` + reader-first banner. Old card stays; new card is current truth.

`pnpm docs:amend` HARD-FAILs on `docs/cards/**` paths and on `tier: card` frontmatter (defense in depth).

## Steward loop (weekly)

- `.github/workflows/docs-steward.yml` — Mondays 09:00 UTC + manual `workflow_dispatch`.
- Output: GitHub Issue titled "Docs Steward report — YYYY-MM-DD" with labels `docs-steward` + `automated`.
- `/docs-steward` slash command triggers a fresh local run + triage table (read-only proposal mode).
- Thresholds: stale >30 commits OR >90 days + doc touched; untouched-but-referenced >=10 inbound + >60 days; citation-drift cap 5 file:line per doc.
- Env-var toggles: `STEWARD_SKIP_CITATION_DRIFT=1`, `STEWARD_SKIP_INBOUND=1`.

## Validator scoping

- `scripts/check-dead-links.sh` + `scripts/check-diagram-health.sh` exclude `docs/cards/**` AND `docs/archive/**` as link sources. Cards are queryable (frontmatter + wikilink graph), not navigated by prose links — see [[feedback_cards_queryable_not_navigated]].
- Cards ARE still frontmatter-validated (`pnpm docs:check` counts 817 docs total).
- Anti-pattern HARD-FAIL (`*-amendment-*.md` etc.) stays scoped to `docs/superpowers/specs/`; cards unaffected.
