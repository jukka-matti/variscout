This audit deliberately overrides the `maintaining-documentation` skill's prior rule ("delivered specs stay as living references"). Rationale: at 64 specs the living-spec policy costs more than it gains. New default post-Phase-5 = **archive-on-delivery unless an ADR is missing or the spec is still actively informing work**. Skill will be updated in the same PR.

## Rules applied

Priority order — first matching rule wins:

1. Process-flow exception → **KEEP** (only-source-of-truth).
2. `status ∈ {superseded, archived}` → **ARCHIVE** (status alone; inbound refs get rewritten at execution).
3. `strong inbound ≥ 2` (code/agent/skills) → **KEEP**.
4. `delivered AND strong = 0` → **ARCHIVE** (feature shipped; ADR + code are ground truth).
5. `draft AND strong = 0 AND age > 14d` → **ARCHIVE** (abandoned draft).
6. Everything else → **REVIEW**.

Index files (`docs/07-decisions/index.md`, `docs/superpowers/specs/index.md`) are excluded as sources — they are bookkeeping, not load-bearing.

Columns: Code, Agent, Skills = strong buckets. Peers = other ADRs/specs. Docs = broader `docs/` tree.
