# VariScout — Decision Log

This is the durable home for decisions that would otherwise live only in plan files. **Before re-opening any topic, read this log first.** Every session that defers a feature, supersedes a design, or opens a new question is expected to touch this document.

The log has five sections:

1. **Replayed Decisions** — closed calls we keep relitigating; pinned so future sessions don't re-derive them.
2. **Open Questions** — known unknowns. Each leaves by becoming a spec / ADR or being dropped.
3. **Named-Future** — features deliberately deferred with intent to remember.
4. **Session Backlog** — running list of conversations / work, with lifecycle state.
5. **User Journey Map** — every major journey, current shipped state vs documented state.

When in doubt: capture, don't invent. Record the decision; link to its source artifact.

- **ADR-073 — No statistical roll-up across heterogeneous units.** Capability indices computed against different `SpecRule`s cannot be combined arithmetically (Watson's locality rule). Enforced by **structural absence**, not by permission predicate: the engine does not expose `meanCapabilityAcrossHubs` or any named variant; CI guard tests in `packages/core/src/__tests__/architecture.noCrossInvestigationAggregation.test.ts` enforce the absence. Distributions, not aggregates. _Closed 2026-04-29._ Source: [`docs/07-decisions/adr-073-no-statistical-rollup-across-heterogeneous-units.md`](07-decisions/adr-073-no-statistical-rollup-across-heterogeneous-units.md). Generalizes to any heterogeneity dimension (different specs, contexts, products, suppliers, shifts) — not just process families. See `feedback_aggregation_heterogeneous_specs.md`.

- **ADR-075 — PWA atomic deploy + update policy.** Four-layer response to a user-reported live-deploy regression where a stale SW-cached `index.html` referenced chunk hashes that no longer existed. (1) SW update mode is `autoUpdate` with `skipWaiting: true`, `clientsClaim: true`, `cleanupOutdatedCaches: true` — auto-claims existing users on next visit (see ADR-075 Amendment 2026-05-02 below for why the original prompt-mode shape was reverted). (2) Every `React.lazy()` site wraps in `lazyWithRetry`, which on chunk-load error evicts `workbox-precache-*` and reloads once-per-session (`sessionStorage` guarded against Safari private mode); `PasteScreen` idle-prefetched so the failure self-heals before user click. (3) Vercel headers: `vercel.live` + Pusher allowed in CSP (`script-src` + `connect-src` + `frame-src`), explicit MIME for `/assets/*`, immutable cache for `/assets/*`, `no-cache` for `/index.html` + `/manifest.webmanifest`. (4) `scripts/check-dist-integrity.mjs` blocks any PWA build whose `dist/index.html` references missing chunks, run inside `scripts/pr-ready-check.sh`. _Closed 2026-05-02._ Source: [`docs/07-decisions/adr-075-pwa-atomic-deploy-and-update-policy.md`](07-decisions/adr-075-pwa-atomic-deploy-and-update-policy.md). Cross-references ADR-058 (Azure-app deployment lifecycle) and ADR-059 (web-first).

- **PWA SW prompt-mode reverted (2026-05-02)** — Reverted to `autoUpdate + skipWaiting:true` two days after ADR-075's prompt-mode introduction. Reason: existing users with the old SW have no path to the new SW's update banner because the banner UI lives in NEW code, but old SW serves cached OLD HTML. `lazyWithRetry` (still active) is the actual defensive layer. ADR-075 amended in place; `UpdatePrompt`, `swUpdates`, and `workbox-window` deleted; `frame-src 'self' https://vercel.live` added to CSP for the Vercel Live preview-comments iframe. _Closed 2026-05-02._ Source: [`docs/07-decisions/adr-075-pwa-atomic-deploy-and-update-policy.md`](07-decisions/adr-075-pwa-atomic-deploy-and-update-policy.md) Amendment.

SCOUT owns L1 outcome reading; FRAME owns L2 authoring; the Hub Capability tab owns L2 reading; Investigation Wall owns L3 hypothesis canvas; Evidence Map owns L3 factor network; INVESTIGATE owns L3 case-building.
