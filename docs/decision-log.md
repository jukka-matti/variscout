# VariScout ‚Äî Decision Log

This is the durable home for decisions that would otherwise live only in plan files. **Before re-opening any topic, read this log first.** Every session that defers a feature, supersedes a design, or opens a new question is expected to touch this document.

The log has five sections:

1. **Replayed Decisions** ‚Äî closed calls we keep relitigating; pinned so future sessions don't re-derive them.
2. **Open Questions** ‚Äî known unknowns. Each leaves by becoming a spec / ADR or being dropped.
3. **Named-Future** ‚Äî features deliberately deferred with intent to remember.
4. **Session Backlog** ‚Äî running list of conversations / work, with lifecycle state.
5. **User Journey Map** ‚Äî every major journey, current shipped state vs documented state.

When in doubt: capture, don't invent. Record the decision; link to its source artifact.

- **2026-05-28 ‚Äî Spec ¬ß3.5 / ¬ß3.5.1 inflection-point binning refined for G1 ship.** `spec edit` (planning): `docs/superpowers/specs/2026-05-26-canvas-connection-journey-design.md` ¬ß3.5 + ¬ß3.5.1 substantively reframed after a three-round brainstorm that crossed (a) external benchmark research (JMP 16's Interactive Binning + Minitab both ship percentiles + Jenks + manual only, deliberately no "auto-suggest" ‚Äî the SPC vendor convention is that bimodal distributions trigger *stratification by an existing factor*, not silent auto-binning of the column under analysis), (b) our own canonical methodology (`docs/03-features/analysis/probability-plot.md` flowchart line 95-105 prescribes "Multiple clusters ‚Üí Stratify by factor"), and (c) our own Phase C roadmap card (`docs/cards/memory/project_probability_plot_roadmap.md` ‚Äî "inflection points = process transitions; differentiating feature vs Minitab").
