# i18n — editing invariants

- **Apps call `registerLocaleLoaders()` at startup.** Tests register their own loaders per file via `import.meta.glob`.
- **No string concatenation** for translated content — use `formatMessage()` with parameters. Intl API does plural/number/date formatting.
- **Typed message catalogs** live in `packages/core/src/i18n/messages/`. Adding a key updates the typed union — all callers remain type-safe.
- **Chinese locales**: files are `zhHans.ts` / `zhHant.ts` (camelCase). `LOCALE_TO_FILENAME` map resolves the camelCase ↔ BCP-47 mismatch. Don't rename.
- **`registerLocale()`** (single-locale direct injection) is exported for tests; prefer `registerLocaleLoaders()` in app code.
- **Display boundary**: `formatStatistic()` from `@variscout/core/i18n` wraps stat values with `Number.isFinite()` guard and locale-aware formatting. Never `.toFixed()` (enforced by ESLint).
