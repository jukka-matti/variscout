# Security Policy

## Supported Versions

Only the latest release of VariScout is supported with security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in VariScout, please report it responsibly.

**Contact:** security@variscout.com

**Please include:**

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

**Do not** open a public GitHub issue for security vulnerabilities.

## Response Timeline

- **Acknowledgement:** within 48 hours
- **Triage:** within 7 days
- **Fix timeline communicated:** within 14 days

## Scope

**In scope:**

- VariScout application code (PWA, Azure App)
- Infrastructure templates (ARM templates)
- CI/CD pipeline configuration

**Out of scope:**

- Azure platform vulnerabilities (report to [Microsoft MSRC](https://msrc.microsoft.com/))
- Third-party dependency vulnerabilities (report to the respective maintainers)
- Social engineering attacks

## Bug Bounty

VariScout does not currently operate a bug bounty program.

## Credit

Security researchers who report valid vulnerabilities will be credited in release notes, with their consent.
